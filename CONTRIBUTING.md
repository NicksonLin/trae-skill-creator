# Contributing

Thanks for your interest in improving `trae-skill-creator`.

## How To Contribute

1. Open an issue describing the problem or proposal
2. Fork the repository and create a feature branch
3. Keep changes focused on a single improvement
4. Run the skill against the existing example cases
5. Add a new eval case if your change addresses a new failure mode
6. Submit a pull request describing the change and the eval case

## Style Guidelines

- Match the existing tone of `SKILL.md`
- Prefer imperative, concrete instructions
- Avoid speculative rules without an eval case
- Update `CHANGELOG.md` for any user-visible change

## Code of Conduct

Be respectful, constructive, and focused on improving skill design quality.
